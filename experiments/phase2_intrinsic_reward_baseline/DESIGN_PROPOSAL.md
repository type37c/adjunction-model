# Phase 2 実験設計に関する考察と提案

**文書バージョン**: 1.0  
**作成日**: 2026-02-18  
**作成者**: Manus AI

## 1. はじめに：3つの未解決の問い

Phase 2の実験設計にあたり、以下の3つの問いが相互に絡み合っていることが明らかになりました。

1.  **「橋渡し」の問い:** Agent Cは価値関数（η軌跡の改善）だけで、どのようにして「アフォーダンスを推測する」能力を獲得するのか？
2.  **「スケール」の問い:** F/Gは、Agent Cによる注意選択（オブジェクト全体／部分）に応じて、意味のあるηを返せるのか？
3.  **「境界」の問い:** F/Gの単純性と、Agent Cへの情報提供の有用性との間の緊張をどう解決し、両者の役割分担をどこに定めるか？

本ドキュメントは、これらの問いを解きほぐし、具体的な設計方針を提案するものです。

## 2. 解きほぐす順序：原理から具体へ

これらの問いは、**3 → 2 → 1**の順で解きほぐすのが最も合理的です。まず最も根源的な「境界」の問いに明確な立場を定め、その決定に基づいて「スケール」の技術的課題を解決し、最後にそのアーキテクチャの上で「橋渡し」の問い（＝評価方法）を定義します。

## 3. Step 1: 「境界」の問いへの回答 → F/Gの役割を固定する

まず、**「F/Gは単純な装置であるべき」**という立場を、Phase 2の設計における**揺るぎない公理**として採用します。

> **公理1: F/Gは文脈を読まない。** F/Gは、入力された点群がオブジェクト全体なのか部分なのかを知らないし、知る必要もありません。F/Gの責務は、与えられた点群に対して「再構成誤差η」と「最も近い既知のアフォーダンス」を返すことだけに限定されます。

この公理を置くことで、「境界」の問いに関する迷いを断ち切ります。F/Gは賢くなりません。汎化の責務は100%、Agent Cが負います。

## 4. Step 2: 「スケール」の問いへの回答 → Agent Cの仕事として解決する

F/Gが単純であるという公理から、「スケール」の問いの答えは自ずと導かれます。**大局と部分の比較・解釈は、Agent Cの仕事**です。

### 提案するメカニズム：比較η（Comparative η）

Agent Cは、オブジェクト全体を見たときのη（**η_whole**）と、オブジェクトの一部に注目したときのη（**η_part**）の両方を観測します。そして、この**2つのηの関係性そのもの**を、新しい情報源として利用します。

| 状況 | η_whole | η_part | Agent Cの解釈（創発される意味） |
| :--- | :--- | :--- | :--- |
| **既知の形状**<br>（例：立方体全体を見る vs その一面を見る） | 低い | 低い | 「この部分は、全体と整合性が取れている。既知の構造だ」 |
| **未知の組み合わせ**<br>（例：取っ手のついた立方体 vs 取っ手部分を見る） | 高い | 低い | 「全体としては未知だが、この部分は既知の形状（円柱）に似ている。ここに注目すれば理解できそうだ」 |
| **ノイズや無関係な部分**<br>（例：立方体全体を見る vs 床の模様を見る） | 低い | 高い | 「この部分に注目すると、かえって理解が難しくなる。無視すべき情報だ」 |

このメカニズムにより、F/Gは単純なままで、Agent Cは「どこに注目すれば世界をより良く理解できるか（＝η_partがη_wholeより改善するか）」を学習できるようになります。注意選択は、この**比較ηを改善するための行動**として、価値関数を通じて自然に学習されます。

## 5. Step 3: 「橋渡し」の問いへの回答 → 「推測」を定義し、評価する

上記メカニズムを導入することで、「アフォーダンスを推測する」という行為を、**観測可能で評価可能なプロセス**として定義できます。

### 「アフォーダンスの推測」の操作的定義

> **定義:** エージェントが未知のオブジェクト（η_wholeが高い）に遭遇した際、能動的な注意選択（複数回のη_partの観測）を通じて、最終的にそのオブジェクトに対する適切な行動（例：取っ手を掴む）を選択できた場合、エージェントは「アフォーダンスを推測した」とみなす。

これは、単にηを下げること以上の意味を持ちます。エージェントは、**「情報を集め（注意選択）、仮説を立て（各部分のアフォーダンス予測）、統合して判断する」**という一連の認知プロセスを実行していることになります。

### Phase 2での評価方法

1.  **汎化能力検証データセット:** 訓練データにはない形状の組み合わせ（例：取っ手のついた立方体、複数のオブジェクトが積み重なった状態）を用意します。
2.  **成功の定義:** エージェントが、この未知のオブジェクトに対して、人間が妥当と判断するアフォーダンス（例：「取っ手を掴む」「上のオブジェクトをどかす」）に対応する行動を自発的に選択できれば成功とします。
3.  **ログ分析:** 成功／失敗事例において、エージェントの注意がどこに向けられ、η_wholeとη_partがどのように推移したかを分析し、「推測」のプロセスが設計通りに機能しているかを確認します。

## 6. 結論と次のステップ

3つの問いは、**「F/Gの役割を単純な『測定器』に固定する」**という一つの公理を置くことで、連鎖的に解きほぐすことができます。

-   **境界:** F/Gは測定器、Agent Cが科学者。
-   **スケール:** Agent Cが全体と部分の測定値を比較・解釈する。
-   **橋渡し:** 「推測」とは、Agent Cが測定器を能動的に使って情報を集め、判断するプロセスそのものである。

この設計方針に基づき、Phase 2の実験を**「Agent Cが、単純なF/Gを使いこなし、比較ηを手がかりに注意選択を学習し、未知の形状のアフォーダンスを推測できるか」**を検証する実験として具体化することを提案します。
