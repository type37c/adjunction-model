# Slackの具体例と世界モデルへの汎用性

**Date**: 2026-02-15

---

## 1. Slackとは具体的に何がズレているのか

Slackは、随伴 F ⊣ G の不完全性を測る2つの指標、η (unit slack) と ε (counit slack) から成る。

### η (unit slack): 知覚的余白

ηは「形状を見て、行動可能性を推論し、その行動可能性から形状を再構成したときのズレ」である。

**例：コップを見る**

1.  **F (知覚)**: コップの形状を見る → 「掴める」「液体を注げる」というアフォーダンスを推論
2.  **G (再構成)**: 「掴める」「液体を注げる」から形状を逆推論 → 「円筒形で、上が開いている何か」

ηは、**元のコップ**と**再構成された「何か」**のズレに相当する。このズレは、Fの段階で**情報が損失**することに起因する。コップの色、重さ、材質、模様といった情報は、「掴める」「注げる」というアフォーダンスには反映されない。Gがどれだけ頑張っても、Fが捨てた情報は復元できない。

> **ηは「目的のフィルターが捨てた情報の量」を測る。**

### ε (counit slack): 意味的余白

εは逆方向のズレであり、「行動可能性から形状を推論し、その形状から行動可能性を再推論したときのズレ」である。

**例：「掴める」という行動可能性から出発する**

1.  **G (想像)**: 「掴める」→ 典型的な掴めるもの（コップ、ペン、ドアノブ...）の形状を推論
2.  **F (再解釈)**: その形状を見て → 「掴める」だけでなく「投げられる」「回せる」「書ける」...

εは、**元の「掴める」**と**再推論された行動可能性の集合**のズレに相当する。ここで重要なのは、εは**増える方向にズレる**という点である。「掴める」から出発したのに、「投げられる」「回せる」という**新しい行動可能性が創発**している。これがPhase 2実験でεが+417,837%という驚異的な増幅を示した理由である。

> **εは「意味の再解釈によって生まれる新しい可能性の量」を測る。**

### ηとεの非対称性

| | η (unit slack) | ε (counit slack) |
|:---|:---|:---|
| **方向** | 形→行動→形 | 行動→形→行動 |
| **ズレの性質** | 情報の損失（知覚的余白） | 意味の増殖（意味的余白） |
| **上限** | ある（元の形状の情報量が上限） | ない（再解釈は無限に広がりうる） |
| **Phase 2実験** | +14% | +417,837% |

## 2. 世界モデルへの汎用性

現在の実装では、Slackは3次元点群と変位ベクトルという連続空間上のL2ノルムで計算されている。これは現在の実験設定では有効だが、汎用的な世界モデルには適用できない。

**しかし、Slackの「概念」は汎用的である。計算方法が汎用的でないだけだ。**

汎用化の道筋は、FとGの入出力空間を一般化し、Slackの計算を潜在空間に移すことである。

1.  **形状空間の一般化**: 点群 → 任意のモダリティの潜在表現 `z_shape`（画像、音声、テキスト、グラフ等）
2.  **行動空間の一般化**: 変位ベクトル → 任意の行動の潜在表現 `z_action`（連続、離散、階層的）
3.  **Slack計算の一般化**: L2ノルム → 学習された距離関数 `d_S`, `d_A`

```python
# 潜在空間におけるSlackの計算
eta = d_S(z_shape, G(F(z_shape)))    # 形状の潜在空間での距離
epsilon = d_A(z_action, F(G(z_action)))  # 行動の潜在空間での距離
```

この構造は、現在のアーキテクチャと完全に互換性がある。FとGの入出力を潜在表現に変えるだけで、Agent Cの構造、Slackの管理メカニズム、有能感駆動行動といった核心部分はすべてそのまま機能する。

## 結論

現在の実験は、**Slackという概念が機能することの証明**である。世界モデルへの汎用化は、FとGの入出力空間を拡張し、Slackの計算を潜在空間に移すことで実現可能である。

重要なのは、**Agent Cの構造（随伴の選択、Slackの管理、有能感駆動）は変更不要**だということだ。これは、「川床と水流」の比喩そのものである。FとGという川床の形は、流れる水（モダリティ）によって変わる。しかし、水が川床を流れるという**力学**は変わらない。
