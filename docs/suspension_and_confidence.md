# 保留構造と確信度：余白の創発

## 概要

本ドキュメントは、プロジェクトにおける思考プロセスの観察から得られた、保留構造の本質に関する深い洞察を記録する。「これはこうだという断定を一回壊してまた未確定にする」という繰り返しのプロセスが、保留や余白の正体であり、この構造を計算モデルとして実装する可能性について議論する。

## 1. 思考プロセスとしての保留：断定と未確定の往復

### 1.1 観察された思考パターン

このプロジェクトでの思考を振り返ると、以下のようなパターンが見られる：

「これはこうだという断定を一回壊してまた未確定にしたりすることの繰り返し」

この観察は、保留構造の本質に迫る重要な洞察である。

### 1.2 プロジェクト履歴における断定と未確定の実例

**これ自体が保留構造の実演である。**

## 2. 核/エッジに余白を持たせる：結びつきへの余白

### 2.1 2つの示唆的な表現

1. **「核/エッジに余白を持たせる」**
2. **「結びつきに余白を持たせる」**

これらの表現は、保留構造の実装において極めて示唆的である。

### 2.2 現状のモデルとの対応

```
Shape ──[F]──→ Action ──[G]──→ Shape'
         ↑                ↑
       結びつき         結びつき
```

**現状**：FもGも「確定した写像」として学習されている

**問い**：結びつき自体に余白を持たせられないか？

### 2.3 数学的解釈：随伴におけるズレ

随伴 F ⊣ G において：
- **unit η**: Shape → G(F(Shape))
- **counit ε**: F(G(Action)) → Action

この η と ε は「ズレ」を表す。

**現状**：このズレを「エラー」として最小化している

**提案**：ηとεを「余白」として保持することが、保留構造の本質？

## 3. 保留の創発：確信度という新しい次元

### 3.1 創発の条件

保留が創発するためには、以下の3つの条件が必要である：

1. **「確定」と「未確定」の両方の状態が可能であること**
   - 現状：常に確定（重みは固定値）
   - 必要：「この結びつきは暫定的」という状態

2. **「確定を壊す」トリガーがあること**
   - 現状：coherence breakdownがトリガー
   - これは既にある！

3. **「未確定のまま保持する」能力があること**
   - 現状：breakdownがあれば即座に適応
   - **ここが欠けている**

### 3.2 実装のアイデア：確信度の導入

```python
# 現状：重みは確定値
weight = 0.5

# 提案：重みに「確信度」を付ける
weight = 0.5
confidence = 0.8  # 高い = 確定、低い = 余白

# 適応のロジック
if coherence_breakdown and confidence > threshold:
    # 確定していたものが壊れた → 未確定にする
    confidence *= decay
    # 重みは変えない、ただ「信じなくなる」

elif confidence < threshold:
    # 未確定のまま保持（保留）
    # 新しい情報を待つ
    pass
```

これは「ベイズ的な重み」に近いが、より本質的には：

**「学習」ではなく「確信度の調整」として適応を捉える**

## 4. 思考プロセスが示唆すること

### 4.1 保留の3つの状態

人間の思考プロセスにおいて：

1. 「Pは不要」と**断定**した
2. 実験結果を見て、その断定を**壊した**
3. 「やはりPが必要かも」と**未確定にした**
4. しかしまだ完全には確定していない

**この「3」の状態が保留である。**

### 4.2 現状のAgent Cとの違い

現状のAgent Cには「3」がない。breakdownがあれば即座に「4」（新しい確定）に向かう。

## 5. 提案：Agent Cに確信度の次元を追加

### 5.1 新しいAgent State

```
Agent State C = (h, z, confidence)
```

- **h**: 記憶（何を経験したか）
- **z**: 信念（世界はどうなっているか）
- **confidence**: 確信度（その信念をどれだけ信じているか）

### 5.2 confidenceが低いときの振る舞い

Agent Cは：
- F/Gを強く調整しない（FiLMのscale/shift が 1.0/0.0 に近づく）
- 新しい情報を待つ
- **保留状態**

## 6. 結論：保留構造の本質

### 6.1 保留とは何か

保留とは、以下の能力を持つ構造である：

1. **「確定を壊して未確定にする」ことができる**
2. **「未確定のまま保持する」ことができる**

### 6.2 実現に必要なもの

1. 重みではなく「**確信度**」で考える
2. breakdown → 即適応 ではなく → **まず確信度を下げる**
3. 確信度が低いまま一定期間**保持できる**

### 6.3 設計と創発の分離

この「確信度」の次元を追加することで、保留の創発が可能になる。

そして、この設計は「**枠組みは設計し、内容は創発させる**」原則と整合する：

| 設計 | 創発 |
|:----------------------|:-----------------|
| 確信度という次元の存在 | どの結びつきにどれだけ確信を持つか |
| 確信度が下がる条件（breakdown） | どこまで下がったら「保留」とみなすか |
| 確信度が回復する条件（coherence改善） | どの経験から確信を回復するか |
